CV Site Lab is my personal CV and portfolio site, built as more than a static resume page. I treat it as a small production-like engineering project with structured content, bilingual pages, project notes, a technical blog, SEO hygiene, and repeatable build checks.

The goal is to make the site useful as both a professional profile and a visible record of engineering practice: how decisions are made, how scope is controlled, how changes are verified, and how the project evolves without turning into an unreviewable rewrite.

## Engineering Focus

- Static-first Astro site with MDX content and Astro content collections.
- Clear separation between profile pages, project descriptions, and the blog.
- Bilingual structure, with Russian content written first and English content adapted for an international reader.
- SEO basics such as canonical URLs, alternate links, structured data, and useful page descriptions.
- Reviewable workflow based on small diffs, explicit assumptions, `npm run check`, and production builds.

## Operational Angle

Even though this is a personal site, I approach it with production habits: configuration should be explicit, builds should be reproducible, deployment should be predictable, and changes should be easy to review and roll back.

The blog is part of that approach. It will document the actual process of building the site: why a stack was chosen, which trade-offs appeared, what broke, how changes were verified, and which operational concerns still matter for a small public web project.
